# SkillMint Premium UX Direction

**Status:** Direction for beta readiness, not a full redesign
**Principle:** Trust beats flash.

## Desired Feel

SkillMint should feel:

- Premium
- Calm
- Career-focused
- Trustworthy
- Editorial
- Analytical
- Direct

SkillMint should not feel:

- Arcade-like
- Crypto dashboard-like
- Noisy
- Neon-heavy
- Over-gamified
- Fake AI hype

## Theme Direction

Long term, SkillMint should move toward a light-first premium career report direction. Do not migrate the whole app to light theme during this block.

Near-term dark theme rules:

- Reduce tacky glow-heavy treatments in core report areas.
- Use restrained accent colors.
- Keep dark surfaces neutral and readable.
- Make score hierarchy clearer than decoration.
- Keep accessibility and contrast visible in every score card.
- Prefer editorial report-card language over hype language.

## Core Report Areas

Dashboard, resume history, and score explanations should feel screenshot-friendly:

- Fewer stronger cards.
- Clear score meaning and limitation language.
- Stable mobile layouts.
- Long filenames and labels must wrap safely.
- Profile-fit roles and Latest JD Match must stay visually and verbally separate.
- Proof Confidence must always mention evidence candidates when there is space.

## Gamification Direction

SkillMint can use mature progress, but not childish XP.

Use:

- Missions
- Proof completion
- Re-score loop
- Roadmap execution
- Evidence gap closure

Avoid:

- Coins
- Leaderboards
- Arcade badges
- Streak pressure without real career value
- Celebratory copy that hides weak signals

## Comeback Loop

Users should return because:

- They learned what is weak.
- They have missions to improve.
- They can re-score after changing their resume.
- They can test against a new JD.
- They can see proof gaps closing over time.

## Block 1 Final Light UI Audit

**Date:** 2026-07-10
**Roadmap block:** Premium Product UI System

### Findings

- Authenticated shell used a dark radial background, dark sidebar, glow active state, and white text, which made the app feel closer to a neon dashboard than a premium career report.
- Dashboard report cards used dark gradients, black inset cards, violet/blue accents, and several dense grids that reduced readability.
- Proof Evidence Breakdown used a five-column desktop grid that could create an empty gray-cell feeling and cramped evidence cards.
- Harsh Truth used five equal cards, which made direct feedback feel crowded instead of trustworthy.
- Shareable Snapshot was functional but too plain to feel share-worthy.
- Resume history already preserved saved versus active report logic, but the active saved report needed stronger visual priority and safer wrapping.
- ATS/JD page already had one-JD logic and history behavior, but the old styling made it feel like a generic dashboard.
- Roadmap missions were display-only as required, but dark cards and badge styling risked feeling game-like.
- Settings copy correctly described browser-only clear workspace, but the destructive card needed a clearer light trust treatment.
- Upload/setup/auth surfaces used dark first-run visuals that no longer matched the product direction.
- Landing page had a mixed treatment: light shell pieces, but a dark product preview, raw FAQ cards, and a dark CTA that did not match the product experience.

### Light-First Premium Decision

Block 1 moves the visible product toward a light-first premium report system: warm off-white app background, white cards, slate text, subtle slate borders, soft shadows, emerald primary action, amber warnings, rose destructive states, and limited blue/violet.

The product should now read as a serious career report, not a dark admin dashboard or AI template.

### What Changed

- Added `src/components/ui/premium.ts` with small class constants for page stack, hero surfaces, cards, insets, inputs, CTAs, badges, warning/danger states, and dividers.
- Converted authenticated shell, sidebar, topbar, dashboard report surfaces, resume, ATS, roadmap, settings, upload, setup, login/signup auth shell, shared account/action/onboarding cards, and active dashboard visuals to light-first styling.
- Reworked Proof Evidence Breakdown to render real evidence cards in responsive 1/2/3-column layouts instead of a five-column grid.
- Reworked Harsh Truth into a readable summary plus responsive insight grid.
- Upgraded Shareable Snapshot into a compact report-card style with Generated by SkillMint badge and native Share summary action.
- Improved mobile wrapping for filenames, buttons, badges, history cards, and action rows.
- Finished landing light consistency across navigation, hero, problem, product preview, product loop, features, FAQ, CTA, footer, and unused landing components.

### What Intentionally Did Not Change

- Scoring math.
- Proof Confidence math.
- ATS/JD match logic.
- Role matching logic.
- Roadmap generation logic.
- Auth logic.
- Database schema.
- Package dependencies.
- Payments, checkout, analytics implementation, AI chat, job board, account deletion, saved resume deletion, mission completion, active target workflow, or saved JD workflow expansion.
- New landing claims, fake testimonials, fake ratings, fake company names, or fake placement proof.

### Known Visual Debt

- Auth recovery screens are aligned with the light auth shell; future polish should stay limited to copy and edge-state refinement.
- A future token migration could replace remaining raw Tailwind class strings with a more complete product token layer.
- Some legacy dashboard components are updated but still not part of the active dashboard path.

### Future Share Export Note

True PNG/image export for Shareable Snapshot was deferred. The card now visually behaves like an image-style report snapshot, but actual export should be implemented later only if it can be done without risky dependencies or bundle impact.
