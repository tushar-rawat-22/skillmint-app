# UX-005 — Interaction System

Document ID: UX-005

Version: 1.0

Status: Approved

---

# Philosophy

Every interaction should reduce friction.

Every click must create progress.

The interface should feel alive without becoming distracting.

---

# Core Principle

Users should never ask:

"What does this mean?"

Instead they should ask:

"What can I improve next?"

---

# Interaction Hierarchy

Level 1

Primary Action

Only one exists per screen.

Examples

Start Today's Mission

Upload Resume

Continue

Upgrade

---

Level 2

Supporting Actions

View Details

Compare Resume

History

Profile

---

Level 3

Passive Information

Charts

History

Statistics

Analytics

---

# Workspace Rule

The Career Workspace must always display

One Mission

One Progress Indicator

One AI Insight

One Achievement

Nothing else competes for attention.

---

# Progressive Disclosure

Complex information must remain hidden until requested.

Example

Career IQ

↓

Click

↓

Detailed analytics

↓

Historical trend

↓

Skill contribution

Never show all data by default.

---

# CTA Rules

Every page

Exactly one primary CTA.

Never multiple competing buttons.

---

# Interaction Feedback

Every action receives feedback.

Resume Uploaded

Animation

Mission Completed

Celebration

Career IQ Increased

Counter animation

Recommendation Completed

Success notification

---

# Motion Guidelines

Motion should communicate

State

Progress

Completion

Never decoration.

---

# Animation Duration

Hover

120–180ms

Cards

220ms

Dialogs

250ms

Page transitions

350ms

Achievement

700ms

---

# Touch Targets

Minimum

44x44 pixels

---

# Keyboard Support

Every primary action

Keyboard accessible.

---

# Accessibility

Never rely on color alone.

Icons

Text

Motion

must reinforce meaning.

---

# Golden Rule

Interactions should create confidence.

Never surprise the user.
