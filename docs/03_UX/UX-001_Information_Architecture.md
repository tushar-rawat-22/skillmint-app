# Purpose

This document defines the complete structural organization of SkillMint.

It specifies every page, module, navigation layer, and information hierarchy.

No new page may be added without updating this document.

---

# UX Philosophy

SkillMint is not a dashboard.

SkillMint is a guided career operating system.

Users should always know:

• Where they are

• Why they are here

• What they should do next

At no point should users feel lost.

# Information Priority

Priority 1

What needs attention today?

Current employability.

Priority 3

AI recommendations.

Priority 4

Historical information.

Priority 5

Settings.

Users should never need more than one click to reach Priority 1.

---

# UX Rules

Rule 1

Maximum navigation depth:

3 levels.

Rule 2

Every page has exactly one primary action.

Rule 3

No page should require horizontal scrolling.

Rule 4

The most important information must always appear above the fold.

Rule 5

Every chart must answer a question.

Never display decorative analytics.

Rule 6

No feature may exist without a clear JTBD.
