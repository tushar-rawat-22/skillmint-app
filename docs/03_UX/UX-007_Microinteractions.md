# UX-007 — Microinteractions

Document ID: UX-007

Version: 1.0

Status: Approved

---

# Purpose

Microinteractions make SkillMint feel responsive, trustworthy and premium.

They should reinforce progress—not distract from it.

---

# Philosophy

Every animation answers one question:

"What just happened?"

Animations never exist purely for decoration.

---

# Motion Principles

• Fast
• Predictable
• Meaningful
• Interruptible
• Accessible

---

# Hover States

Cards

- Slight elevation
- Border emphasis
- Soft shadow increase

Buttons

- Background transition
- Cursor feedback

Navigation

- Active indicator slides smoothly

---

# Click Feedback

Every click receives immediate feedback.

Examples

- Button compresses slightly
- Ripple avoided
- Loading state begins within 100ms

---

# Page Transitions

Maximum duration

300–350ms

Never block user interaction.

---

# Mission Completion

Sequence

Checkmark

↓

Progress updates

↓

Career IQ animates

↓

Achievement appears

↓

Mission replaced

Total duration

< 1 second

---

# Career IQ Updates

Old Value

↓

Count animation

↓

New Value

↓

Small glow

↓

Return to normal

---

# AI Processing

Never show a spinner alone.

Display live stages:

• Reading Resume

• Understanding Experience

• Mapping Skills

• Calculating Career IQ

• Building Recommendations

Progress should feel alive.

---

# Success Feedback

Minor Success

Green accent

Small toast

Metric animation

Major Success

Celebration card

Achievement unlock

Optional confetti

Confetti is reserved for major milestones only.

---

# Error Feedback

Shake animation

Brief highlight

Clear recovery path

Never punish users.

---

# Notification Behaviour

Notifications slide gently.

Disappear automatically.

Never cover primary content.

---

# Empty States

Illustration

↓

One sentence

↓

Primary CTA

Never leave blank screens.

---

# Motion Accessibility

Respect reduced-motion preferences.

Replace movement with fades when required.

---

# Golden Rule

Motion should increase clarity.

If removing an animation improves usability,

remove the animation.
