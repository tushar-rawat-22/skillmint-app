# SkillMint v1.2 — Account Persistence Freeze

**Release Name:** SkillMint v1.2 — Account Persistence Freeze

**Date:** 2026-07-03

**Status:** Frozen for Sprint 6

---

## Summary

Sprint 6 turns SkillMint from a browser-only Career Operating System into an account-aware product with Supabase-backed persistence. Users can still use localStorage flows without signing in, while signed-in users can save profile details, resume analyses, job matches, rewrite plans, and generated roadmaps to their account.

This release is a stabilization and visibility freeze for account persistence before moving into onboarding, beta UX, and launch readiness.

## User Flows Now Working

- Users can open SkillMint without Supabase credentials configured.
- Users can create an account, log in, and log out when Supabase is configured.
- Signed-in users can save and update a basic persistent profile.
- Resume analysis still saves locally and also attempts account sync.
- The resume page can restore the latest synced resume analysis when localStorage is empty.
- ATS match results, improvement plans, and rewrite plans still save locally and also attempt account sync.
- The ATS page can restore recent synced job matches when local history is empty.
- Roadmaps still generate locally and sync into the latest persisted job match when possible.
- Dashboard and settings show account persistence visibility through the account overview.

---

## Supabase Tables Used

- `profiles`
- `resume_analyses`
- `job_matches`

The `career_snapshots` table exists in the Sprint 6 schema draft but is not actively written by the app yet.

---

## LocalStorage Fallback Behavior

LocalStorage remains active for the core product flows:

- `skillmint:resume-analysis`
- `skillmint:resume-sync-status`
- `skillmint:jd-match`
- `skillmint:jd-match-history`
- `skillmint:jd-match-sync-status`

If Supabase env vars are missing, the user is signed out, the schema has not been installed, or RLS blocks a request, SkillMint keeps the local workflow usable and shows non-blocking sync status messages.

## Known Limitations

- A real production Supabase project must be configured manually.
- `supabase/schema_v1.sql` must be run manually in the Supabase SQL editor.
- OAuth login is not implemented yet.
- Password reset flow is not implemented yet.
- Email template customization is not done yet.
- Server-side protected routes are not implemented yet.
- LocalStorage fallback still exists.
- Production deployment is not done yet.
- Billing and monetization are not implemented yet.
